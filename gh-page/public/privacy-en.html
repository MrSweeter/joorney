Privacy Policy

Last updated: July 3, 2026

This page describes the data that the Chrome extension Joorney may use or store in order to function properly.

This policy applies to the Joorney extension and any related browser-based features it provides.

This privacy policy is publicly available and is linked in the Chrome Web Store developer dashboard privacy field.

1. Data collected

Joorney does not collect personal data for analytics or tracking. The extension uses information only to provide its features and improve your Odoo experience.

1.1 Browsing data and URLs

Joorney may read the active tab URL and web pages opened in the browser to detect whether you are on an Odoo site, enable relevant features described on the Chrome Web Store listing, and provide contextual actions. This browsing activity is collected only for the user-facing Odoo functionality of the extension.

1.2 Odoo session data

When you use Joorney on an Odoo site, the extension may read information present on the Odoo page, such as the internal session object (odoo.__session_info__), to determine your session state and offer useful actions. This data is used only locally in your browser to make the extension work.

1.3 Cookies

Joorney uses cookie access only for the Odoo theme switch feature. It reads and writes the following cookies on the Odoo domain:

  • color_scheme
  • configured_color_scheme

1.4 Local and synced storage data

The extension stores your preferences, feature settings, and cache data locally in chrome.storage.local and chrome.storage.sync. These data may include, for example:

  • Feature enablement preferences
  • Disabled origins
  • Cache data to speed up internal calls
  • Text and user interface settings

1.5 Location data

Joorney may access your geographic location only if you explicitly click the location button in the extension theme settings. This is used only to calculate sunrise and sunset times for automatic dark mode.

1.6 External requests

Joorney makes network requests to the following services:

  • GitHub to retrieve public announcement and version data.
  • SunriseSunset.io to calculate sunrise and sunset times when the corresponding option is enabled.

When automatic dark mode with location is enabled, Joorney may send your latitude and longitude to SunriseSunset.io in order to calculate sunrise and sunset times. This location data is used only for that feature and is not retained for other purposes.

Other service requests do not involve user-identifiable personal data. Joorney does not share user personal data with any third party for advertising, analytics, or profiling.

2. Data use

Data is used only to:

  • Determine whether you are browsing an Odoo site
  • Enable or disable features appropriate for the site
  • Access Odoo APIs at the same time as you, to provide actions such as opening menus or reading data
  • Keep your extension settings and preferences
  • Calculate theme scheduling when you enable the location option

All use of user data is limited to the functionality described here and in the extension listing. Joorney does not use user data for advertising, profiling, or credit decisions.

Joorney does not sell, rent, or share your personal data with third parties.

3. Retention and deletion

Settings and cache data remain stored while the extension is installed. You can delete these data by clearing extension storage from Chrome settings or uninstalling Joorney.

4. No external tracking

Joorney does not use third-party analytics or tracking services (such as Google Analytics, Sentry, Amplitude, Mixpanel, Hotjar, etc.).

5. Required Chrome permissions

The extension manifest declares the following permissions. These permissions are used only for the features described in this policy and in the Chrome Web Store listing.

  • tabs: to read the active tab URL and respond to tab events.
  • webNavigation: to detect navigation changes and dynamically load features.
  • cookies: to read and write Odoo theme-related cookies.
  • storage: to keep your preferences and cache data.
  • contextMenus: to create browser context menus.
  • webRequest: to detect the status of HTTP requests used by some features.
